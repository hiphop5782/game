# 기능명세

## 로컬 유저/플레이 기록

### 목적

서버 없이 같은 기기 안에서 플레이어별 기록을 저장하고 확인하는 기능입니다. 저장소는 브라우저 `localStorage`입니다.

### 저장 키

```js
miniGamePlayRecords.v1
```

### 데이터 구조

```js
{
  users: [
    {
      id: "user_...",
      name: "게스트",
      createdAt: "2026-06-17T..."
    }
  ],
  activeUserId: "user_...",
  records: [
    {
      id: "record_...",
      userId: "user_...",
      gameType: "maze",
      gameName: "미로찾기",
      difficulty: "어려움",
      startedAt: "2026-06-17T...",
      endedAt: "2026-06-17T...",
      durationSec: 123,
      success: true,
      score: 100,
      hintsUsed: 1,
      stats: {}
    }
  ]
}
```

### 화면

홈 화면 상단에 프로필 패널이 있습니다.

- 현재 플레이어 표시
- 프로필 선택
- 새 이름 입력
- 추가 버튼
- 기록 보기 버튼

`records-view` 화면에서는 현재 플레이어의 기록만 보여줍니다.

요약 카드:

- 전체 플레이
- 성공 기록
- 힌트 사용
- 미로 최고 기록

### 기록 저장 방식

각 게임은 완료 시 `gameRecords.recordGameResult(record)`를 호출합니다. 공통 필드는 `gameRecords`가 보정하고, 게임별 상세값은 `stats`에 담습니다.

## 미로찾기

### 목적

랜덤 미로를 생성하고, 플레이어가 입구에서 출구까지 길을 찾아 탈출하는 게임입니다.

### 사용자 흐름

1. 홈에서 `미로찾기` 선택
2. 난이도 선택
3. 방향키 또는 화면 버튼으로 이동
4. 막히면 힌트 사용
5. 출구에 도착하면 탈출 성공
6. `새 미로`로 다시 시작

### 난이도

| 난이도 | 크기 | 힌트 | 특징 |
| --- | --- | --- | --- |
| 쉬움 | 11 x 11 | 4 | 짧고 단순 |
| 보통 | 15 x 15 | 3 | 기본 난이도 |
| 어려움 | 21 x 21 | 2 | 갈림길 증가 |
| 매우 어려움 | 31 x 31 | 1 | 큰 미로, 힌트 거의 없음 |

### 미로 생성

현재 미로는 루프 없는 트리형 미로입니다.

핵심 속성:

- 통로가 두 줄로 뭉치지 않도록 임의 벽 제거 방식은 사용하지 않습니다.
- 순환 통로가 생기지 않도록 경로 그래프가 트리 구조를 유지합니다.
- 난이도별 `randomPickRate`로 활성 셀 선택 방식을 조절해 갈림길 체감을 다르게 합니다.
- 출구는 시작점 기준으로 먼 가장자리 경로에 배치합니다.

### 힌트

`mazeGame.findPath()`가 현재 위치에서 출구까지 BFS로 경로를 찾습니다.

힌트 사용 시:

- 남은 힌트가 1 줄어듭니다.
- 다음 몇 칸이 노란색으로 표시됩니다.
- 출구까지 전체 정답을 보여주지는 않습니다.

### 조작

- 키보드 방향키
- 화면의 위/아래/왼쪽/오른쪽 버튼
- 가운데 `?` 버튼은 힌트

## 사진/동영상 퍼즐

### 목적

사용자가 직접 선택한 사진 또는 동영상을 원하는 줄/칸 수로 나누고, 조각을 다시 맞추는 게임입니다. 동영상은 첫 프레임을 퍼즐로 만들고, 완성 후 원본 동영상을 재생합니다.

### 사용자 흐름

1. 홈에서 `사진 퍼즐` 선택
2. 사진 또는 동영상 파일 선택
3. 줄 수 입력
4. 칸 수 입력
5. `퍼즐 만들기` 클릭
6. 오른쪽 조각 영역에서 조각을 드래그
7. 왼쪽 퍼즐판의 슬롯에 배치
8. 막히면 `힌트` 클릭
9. 모든 조각을 맞추면 완성 메시지 표시
10. 동영상 퍼즐인 경우 원본 동영상 플레이어 표시 및 재생 시도

### 입력 제한

- 줄: 2 ~ 8
- 칸: 2 ~ 8

### 조각 렌더링 방식

각 조각은 별도의 이미지 파일로 잘라 저장하지 않습니다.

대신 같은 이미지 URL을 공유하면서:

- `background-image`
- `background-size`
- `background-position`

을 이용해 각 조각 위치만 다르게 보여줍니다.

동영상 파일의 경우:

- `URL.createObjectURL(file)`로 원본 동영상 URL을 만듭니다.
- 숨은 `<video>` 요소가 첫 프레임을 로드합니다.
- `<canvas>`에 첫 프레임을 그립니다.
- `canvas.toDataURL()` 결과를 퍼즐 이미지로 사용합니다.
- 퍼즐 완성 후 원본 동영상 URL을 `<video controls playsinline>`에 넣어 재생합니다.

### 드래그 앤 드롭

PC에서는 드래그 앤 드롭을 사용합니다.

- 조각은 `draggable` 요소입니다.
- 퍼즐판 슬롯과 조각 보관함이 드롭 영역입니다.
- 슬롯에 이미 조각이 있으면 기존 조각은 원래 위치로 이동합니다.
- 힌트로 맞춰진 조각은 `locked` 처리되어 움직일 수 없습니다.

모바일에서는 HTML Drag and Drop이 불안정할 수 있어 선택 배치 방식도 지원합니다.

- 조각을 터치하면 선택됩니다.
- 놓을 칸을 터치하면 해당 칸으로 이동합니다.
- 선택된 조각은 테두리로 표시됩니다.

### 힌트

힌트 버튼을 누르면:

- 아직 맞지 않은 슬롯 하나를 찾습니다.
- 그 슬롯의 정답 조각을 찾아 자동 배치합니다.
- 기존에 잘못 놓인 조각이 있으면 보관함으로 돌려보냅니다.
- 자동 배치된 조각은 잠깁니다.
