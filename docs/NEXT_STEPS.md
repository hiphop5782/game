# 다음 작업 제안

## 우선순위 높음

### 1. `index.html` 분리

현재 `index.html`이 매우 커졌습니다. 유지보수를 위해 아래처럼 분리하는 것이 좋습니다.

- `css/maze.css`
- `css/photo-puzzle.css`
- `js/maze-game.js`
- `js/photo-puzzle-game.js`

단, 기존 코드가 `index.html` 중심으로 많이 작성되어 있으므로 한 번에 전체를 분리하기보다 새 기능부터 천천히 빼는 것이 안전합니다.

### 2. 모바일 퍼즐 조작 고도화

사진 퍼즐은 현재 PC용 HTML Drag and Drop과 모바일용 `조각 선택 -> 칸 터치` 방식을 함께 지원합니다. 실제 모바일 사용 테스트 후 더 자연스러운 조작으로 다듬을 수 있습니다.

개선 방향:

- Pointer Events 기반 직접 드래그 구현
- 조각이 손가락을 따라다니는 시각 효과
- 선택된 조각의 확대 미리보기
- 슬롯 터치 시 교체 애니메이션

### 3. 사진 퍼즐 완성 UX 개선

현재는 완성 메시지만 표시합니다.

추가하면 좋은 것:

- 완성 애니메이션
- 완성 시간 표시
- 이동 횟수 표시
- 다시 섞기 버튼
- 같은 사진으로 난이도 변경

## 우선순위 중간

### 4. 미로 난이도 세부 조정

현재 미로는 루프 없는 트리형 미로입니다. 사용자가 어렵다고 느끼는 정도는 경로 길이, 갈림길 밀도, 힌트 수가 함께 영향을 줍니다.

조정 후보:

- `randomPickRate`
- 난이도별 미로 크기
- 힌트가 보여주는 칸 수
- 출구를 더 먼 위치로 강제하는 조건

### 5. 미로 성공 기록

추가하면 좋은 지표:

- 난이도별 최고 기록
- 최소 걸음 수
- 힌트 사용 횟수
- 완료 시간

브라우저 `localStorage`를 쓰면 서버 없이 저장할 수 있습니다.

### 6. 사진/동영상 퍼즐 미리보기

퍼즐 만들기 전 원본 사진 또는 동영상 첫 프레임 미리보기를 보여주면 사용성이 좋아집니다.

주의:

- 너무 큰 이미지는 렌더링이 느려질 수 있습니다.
- 필요하면 `<canvas>`로 적당한 크기로 줄여 사용하는 방법을 고려합니다.
- 동영상 자동 재생은 모바일 브라우저 정책에 따라 막힐 수 있습니다.

## 우선순위 낮음

### 7. 공통 게임 UI 컴포넌트화

반복되는 구조가 많습니다.

- 게임 헤더
- 점수판
- 액션 버튼
- 피드백 메시지

장기적으로는 공통 CSS/JS 헬퍼로 빼면 좋습니다.

### 8. 접근성 개선

현재 일부 버튼에 `aria-label`이 있으나 전체적으로 일관되지는 않습니다.

개선 후보:

- 키보드 포커스 스타일
- 사진 퍼즐 슬롯 이름
- 드래그 외 대체 조작
- 색상만으로 상태를 구분하지 않기
