# 작업내역

## 1. GitHub 저장소 연결

- `https://github.com/hiphop5782/game` 저장소를 로컬 작업 폴더로 가져왔습니다.
- 기본 HTTPS clone에서 Windows Git TLS/자격 증명 문제가 있어 `http.sslBackend=openssl` 옵션으로 clone을 성공시켰습니다.

## 2. 미로찾기 게임 추가

새 게임 모드 `미로찾기`를 추가했습니다.

추가된 항목:

- 홈 화면 카드 추가
- `maze-view` 화면 추가
- `mazeGame` 객체 추가
- 랜덤 미로 생성
- 키보드 방향키 이동
- 화면 이동 버튼
- 걸음 수 표시
- 탈출 횟수 표시
- 힌트 기능
- 새 미로 버튼

초기 미로는 DFS 기반으로 생성했습니다.

## 3. 미로 시작점/끝점 개선

처음에는 시작점과 끝점이 미로 내부에 있었습니다. 이후 사용자의 요청에 따라 바깥과 연결된 입구/출구 구조로 변경했습니다.

현재 구조:

- 시작점은 바깥 벽에 뚫린 입구입니다.
- 출구도 바깥 벽에 뚫린 칸입니다.
- 출구에 도착하면 탈출 성공으로 처리됩니다.

## 4. 미로 캐릭터 변경

사용자가 넘버블록스 캐릭터 느낌을 요청했습니다.

저작권 있는 실제 캐릭터 이미지는 사용하지 않고, 오리지널 숫자 블록 캐릭터 스타일로 구현했습니다.

현재 캐릭터:

- 빨간 블록 형태
- 숫자 `1` 표시
- 눈과 입체감 스타일
- CSS만으로 렌더링

## 5. 미로 난이도 추가

난이도 선택 기능을 추가했습니다.

현재 난이도:

- 쉬움: `11 x 11`, 힌트 4개
- 보통: `15 x 15`, 힌트 3개
- 어려움: `21 x 21`, 힌트 2개
- 매우 어려움: `31 x 31`, 힌트 1개

난이도 버튼을 누르면 새 미로가 생성됩니다.

## 6. 미로 갈림길 조정

처음에는 미로가 너무 쉽다는 피드백이 있었습니다. 갈림길을 늘리기 위해 벽을 추가로 뚫는 방식이 도입되었으나, 이 방식은 다음 문제가 있었습니다.

- 길이 두 줄로 붙어 보임
- 순환형 통로가 생김
- 전통적인 미로 느낌이 약해짐

이후 방식을 수정했습니다.

현재 방식:

- 나중에 벽을 억지로 뚫지 않습니다.
- 미로 생성 단계에서 활성 셀 선택 방식을 조절합니다.
- 난이도가 높을수록 더 넓게 퍼지는 생성 방식을 사용합니다.
- 통로 그래프는 루프가 없는 트리 구조를 유지합니다.
- 출구는 시작점에서 먼 바깥 가장자리로 배치합니다.

검증 시 `hasCycle: false`로 확인했습니다.

## 7. 사진/동영상 퍼즐 게임 추가

새 게임 모드 `사진 퍼즐`을 추가했습니다. 이후 사진뿐 아니라 동영상도 선택할 수 있도록 확장했습니다.

추가된 항목:

- 홈 화면 카드 추가
- `photo-puzzle-view` 화면 추가
- `photoPuzzleGame` 객체 추가
- 사진 또는 동영상 파일 선택
- 줄/칸 수 입력
- 퍼즐 만들기 버튼
- 드래그 앤 드롭으로 조각 배치
- 모바일용 `조각 선택 -> 칸 터치` 조작
- 맞춘 조각 수 표시
- 힌트 버튼
- 동영상 완성 후 재생 영역

사진 퍼즐은 사용자가 선택한 이미지를 CSS `background-image`, `background-size`, `background-position`으로 조각처럼 보여주는 방식입니다.

동영상 퍼즐은 사용자가 선택한 동영상의 첫 프레임을 `<canvas>`에 캡처해 퍼즐 이미지로 사용합니다. 퍼즐을 모두 맞추면 원본 동영상 플레이어가 표시되고 재생을 시도합니다. 모바일 브라우저의 자동 재생 정책 때문에 자동 재생이 막히면 사용자가 재생 버튼을 누를 수 있도록 메시지를 보여줍니다.

힌트 기능:

- 아직 맞지 않은 슬롯을 찾습니다.
- 해당 슬롯에 맞는 조각을 자동으로 배치합니다.
- 힌트로 배치된 조각은 `locked` 상태가 되어 다시 움직일 수 없습니다.

## 8. 사진 퍼즐 비율 보정

줄/칸 수가 정사각형이 아닐 때 조각이 어색해질 수 있어 보정했습니다.

현재 처리:

- 퍼즐판은 정해진 크기의 격자로 렌더링합니다.
- 조각은 줄/칸 비율에 맞춘 `aspect-ratio`를 사용합니다.
- 보드 안에 들어간 조각은 슬롯 높이에 맞춰 채워집니다.

## 9. 모바일 사진 퍼즐 조작 개선

모바일에서 HTML Drag and Drop이 잘 동작하지 않는 문제가 있어 터치 친화적인 선택 배치 방식을 추가했습니다.

현재 조작:

- PC: 드래그 앤 드롭
- 모바일: 조각 터치 후 놓을 칸 터치
- 선택된 조각은 테두리로 표시
- 힌트로 잠긴 조각은 다시 선택하거나 이동할 수 없음

## 10. 로컬 유저/플레이 기록 추가

서버 없이 브라우저 `localStorage`에 프로필과 플레이 기록을 저장하는 기능을 추가했습니다.

추가된 항목:

- 홈 화면 프로필 패널
- 기본 `게스트` 유저 자동 생성
- 새 유저 이름 추가
- 현재 유저 선택
- `records-view` 기록 화면
- 공통 기록 저장 객체 `gameRecords`
- 미로찾기 완료 기록 저장
- 사진/동영상 퍼즐 완료 기록 저장

기록 저장 키:

```js
miniGamePlayRecords.v1
```

게임별 완료 시 저장되는 주요 값:

- 게임 종류
- 게임 이름
- 난이도
- 시작/종료 시간
- 플레이 시간
- 성공 여부
- 점수
- 힌트 사용 수
- 게임별 상세 `stats`
